templateOptions Type defaultValue Directive Description
label String Primary checkbox name
mdIndeterminate Expression md-indeterminate This determines when the checkbox should be rendered as 'indeterminate'. If a truthy expression or no value is passed in the checkbox renders in the md-indeterminate state. If falsy expression is passed in it just looks like a normal unchecked checkbox. The indeterminate, checked, and unchecked states are mutually exclusive. A box cannot be in any two states at the same time. Adding the 'md-indeterminate' attribute overrides any checked/unchecked rendering logic. When using the 'md-indeterminate' attribute use 'ng-checked' to define rendering logic instead of using 'ng-model'.
mdNoInk Boolean md-no-ink Use of attribute indicates use of ripple ink effects
ngChange String ng-change Angular expression to be executed when input changes due to user interaction with the input element.
ngChecked Expression ng-checked If this expression evaluates as truthy, the 'md-checked' css class is added to the checkbox and it will appear checked.
ngFalseValue Expression ng-false-value The value to which the expression should be set when not selected.
ngTrueValue Expression ng-true-value The value to which the expression should be set when selected.
